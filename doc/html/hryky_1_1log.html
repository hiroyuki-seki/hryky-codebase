logging

This module is intended to output the runtime information. The destinations to which the information will be saved can be file or memory. The elements in the information can be hierarchical structures.

hryky::log::writer::File

hryky::log::writer::File writes messages out to a file. hryky::log::writer::File::operator() can be called thread safely.

Usage of custom class


If you want to use your arbitrary class as the second argument of hryky::reduction::Map::append, you have to define the function to instantiate hryky::reduction_type in the namespace where your class is defined as below.


  hryky::reduction_type NameSpaceOfYourClass::reduce(
    hryky::reduction::name_type const & name, YourClass const & instance)
  {
    ...
  }

In hryky::reduction::Map::append(), an appropriate reduce() is called with the use of C++ ADL(Argument Dependent Lookup).


    // This class can be used as the second argument of hryky::reduction::Map::append().
    class UserValue
    {
    public :
        UserValue(int value) : value_(value) {}
        int value() const { return this->value_; }
    private :
        int value_;
    };



    ((*map)
     .append(
         (*reduction::map("aggregation"))
         .append("int_20",          int(20))
         .append("float_20.0f",     float(20.0f))
         .append("bool_true",       bool(true))
         .append("user_value",      UserValue(4))



hryky::reduction_type hryky::test::reduce(
    reduction::name_type const & name, UserValue const & param) {
    // instantiates reduction_type as key-value pairs.
    reduction::map_type map = reduction::map(name);

    // appends the child pair to the reduction::map_type.
    map->append("value", param.value());

    // returns hryky::reduction_type
    return map;
}
